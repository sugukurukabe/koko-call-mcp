# 入札調査の2時間15分が2分になった話

> note.com 公開用原稿。壁が note.com に貼り付けて公開する。
> 想定読者: 入札担当者・行政書士・中小企業経営者・自治体職員

---

## 最初に結論

官公需入札の調査作業、AIアシスタントに話しかけるだけで全部終わります。

「鹿児島県のIT系入札を探して、うちに合う順に並べて」

これだけで、

1. 全国の入札情報から条件に合う案件を絞り込む
2. 自社条件でAIスコアリング
3. 仕様書PDFを自動取得して要件を整理
4. 参加資格のチェック
5. 締切日をカレンダーに追加

まで終わります。

手作業なら2時間15分かかっていた作業が、約2分です。

## 従来の入札調査

中小企業庁の官公需情報ポータルサイト（KKJ）には、年間180万件超の公共調達情報が掲載されています。

でも実際に使おうとすると、こんな壁があります。

**キーワード検索の精度が低い**。「システム」で検索しても、関係ない案件が大量に混じる。絞り込みに時間がかかる。

**仕様書がPDF**。ダウンロードして開いて読む。参加資格が後ろのページにしか書いていない。20ページ読んで「うちの資格じゃ無理だった」がよくある。

**締切管理が属人的**。Excelで管理している会社が多い。抜け漏れが起きる。

**1件あたりの調査時間**

| 作業 | 手作業 |
|------|--------|
| 案件検索・絞り込み | 30分 |
| 仕様書読み込み | 1時間 |
| 参加資格確認 | 30分 |
| 締切管理登録 | 15分 |
| **合計** | **2時間15分** |

週に10件チェックしたら、22.5時間。それが調査だけでかかっている計算です。

## JP Bids MCPでできること

JP Bids MCPは、このKKJのAPIをAIアシスタントから直接操作できるようにしたツールです。

Claude（Anthropic）やCursor、ChatGPTなどのAIアシスタントに接続すると、日本語で話しかけるだけで入札調査ができます。

**話しかけるだけでいい例**

```
「農林水産省の過去3年の発注傾向を教えて」
```
→ 落札実績取得 → 予算規模・競合・落札率を分析 → 次の入札戦略を提案

```
「この仕様書PDFの参加資格と評価基準を整理して」
```
→ PDF自動取得 → 要件抽出 → 不足書類の取得方法まで提示

```
「来月締切の役務案件を鹿児島県で5件リストアップして」
```
→ 締切日フィルタ → 地域絞り込み → ICSカレンダーで予定登録
