## 日本語

- 基本プロトコルは JSON-RPC 2.0。HTTP transport では `MCP-Protocol-Version` を初期化後のリクエストに含める。
- サーバーは初期化時に実装済み capability だけを宣言する。JP Bids MCP v0.1.0 は `tools`, `resources`, `prompts`, `completions`, `logging` を提供し、`tasks` は日次通知などの長時間処理が入るまで宣言しない。
- Streamable HTTP は `/mcp` 単一エンドポイントで `POST` を処理する。SSE `GET` は実装しない場合 `405` を返せる。
- Tool result は `content` と `structuredContent` を併用する。構造化出力を返すToolは `outputSchema` と実結果の整合性を保つ。
- Tools, Prompts, Resources, Resource Templates の list 系は opaque cursor による pagination に対応する仕様だが、初期の静的リストでは `nextCursor` を返さない。
- Logging は `notifications/message` と RFC 5424 の severity を使い、機密情報や個人情報を含めない。

## English

- The base protocol is JSON-RPC 2.0. HTTP transport requests include `MCP-Protocol-Version` after initialization.
- The server declares only implemented capabilities. JP Bids MCP v0.1.0 exposes `tools`, `resources`, `prompts`, `completions`, and `logging`; `tasks` remains out until long-running workflows exist.
- Streamable HTTP is served from a single `/mcp` endpoint. Unsupported SSE `GET` requests may return `405`.
- Tool results include both `content` and `structuredContent`; tools with structured output keep the result aligned with `outputSchema`.
- List operations support opaque cursor pagination, but the initial static lists return no `nextCursor`.
- Logging uses `notifications/message` and RFC 5424 severity levels without secrets or personal data.

## Bahasa Indonesia

- Protokol dasar adalah JSON-RPC 2.0. Transport HTTP menyertakan `MCP-Protocol-Version` setelah inisialisasi.
- Server hanya mendeklarasikan capability yang benar-benar diimplementasikan. JP Bids MCP v0.1.0 menyediakan `tools`, `resources`, `prompts`, `completions`, dan `logging`; `tasks` belum dideklarasikan sampai ada proses jangka panjang.
- Streamable HTTP dilayani melalui satu endpoint `/mcp`. Permintaan SSE `GET` yang tidak didukung dapat mengembalikan `405`.
- Hasil tool memakai `content` dan `structuredContent`; tool dengan output terstruktur harus konsisten dengan `outputSchema`.
- Operasi list mendukung pagination berbasis opaque cursor, tetapi daftar statis awal tidak mengembalikan `nextCursor`.
- Logging memakai `notifications/message` dan severity RFC 5424 tanpa rahasia atau data pribadi.
