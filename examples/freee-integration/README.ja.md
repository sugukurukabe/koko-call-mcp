# JP Bids MCP × freee MCP — 入札 × 会計の統合ガイド

> **JP Bids MCP** と **freee MCP** は、それぞれ別組織が運営する独立したサーバーです。
> JP Bids MCPはスグクル株式会社が運営し、中小企業庁の官公需（KKJ）公開APIを使用します。
> freee MCPはfreee株式会社が提供し、freee会計・人事労務・請求書APIを使用します。
> 入札・会計に関する意思決定の前に必ず一次資料を確認し、税理士に相談してください。

2つのMCPサーバーで、**入札案件の発見**から**会計処理・請求書発行**までをひとつの会話で完結できます。

| サーバー | データソース | 運営 | 答えてくれること |
|---|---|---|---|
| [JP Bids MCP](https://github.com/sugukurukabe/koko-call-mcp) | KKJ（中小企業庁） | スグクル株式会社 | 「どの入札案件に参加すべきか？」 |
| [freee MCP](https://www.npmjs.com/package/freee-mcp) | freee会計/人事労務/請求書 | freee株式会社 | 「財務状況は？この契約を受けられるか？」 |

---

## セットアップ

### freee MCP（リモート接続）

Claude Desktopの「カスタマイズ」→「カスタムコネクタを追加」で設定：

- 名前: `freee`
- URL: `https://mcp.freee.co.jp/mcp`

### JP Bids MCP（リモート接続）

`https://mcp.bid-jp.com/mcp` にStreamable HTTPで接続。

## ワークフロー

### ワークフロー1：入札前の資金余力確認

**目的**: 入札前に自社の預金残高・売掛金を確認し、資金的に入札可能か判断する。

```
鹿児島県のITシステム案件を探して、スコアの高い案件について、
freeeで当社の直近の売掛金残高と預金残高を確認して、
入札に必要な資金的余力があるか判断してください。
```

### ワークフロー2：落札後の売上取引自動登録

**目的**: 落札後、案件情報をもとにfreeeへ売上取引を自動登録する。

```
案件「2026-KKJ-12345」の詳細を取得して、
落札額と発注機関名をもとに freee に売上取引を登録してください。
勘定科目は「売上高」、取引先は発注機関名で新規作成してください。
```

### ワークフロー3：落札案件からの請求書自動生成

**目的**: 落札案件のメタデータからfreee請求書を自動生成する。

```
落札案件「2026-KKJ-12345」の情報をもとに、
freee請求書で請求書を作成してください。
品名は案件タイトル、金額は落札額、支払期限は納品日から30日後としてください。
```

### ワークフロー4：入札経費の費用対効果分析

**目的**: 入札準備にかかった経費（旅費、印刷費、外注費）と落札実績を照合する。

```
先月の入札関連経費（旅費交通費、印刷費、外注費）をfreeeから集計して、
同期間の落札実績と合わせて、入札の費用対効果を分析してください。
```

## 注意事項

- JP Bids MCPは読み取り専用です。freeeやその他のシステムへの書き込みは行いません。
- freee MCPはOAuth認証が必要です。初回接続時にブラウザでログインが求められます。
- freeeから取得した財務データは御社の機密情報です。LLMの出力を適切に管理してください。
- この連携は参考情報です。入札データに基づく会計処理は必ず税理士に相談してください。
